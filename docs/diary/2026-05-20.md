- **YELLOW 403.1 "over-detection" je iluze.** Začalo jako "1019× detekováno / 47×
  GT = 21× over". Diagnostika přes vizualizaci priority 27 masky odhalila skutečnou
  příčinu: OMAP má **páry area symbolů s identickou RGB**:
  - 401.0 "Open land" (priority 40) ≡ 401.1 "Louka na zelené (upraveno)" (priority 24) → rgb(1.00,0.73,0.21)
  - 403.0 "Rough open land" (priority 43) ≡ 403.1 "Paseka na zelené (upraveno)" (priority 27) → rgb(1.00,0.86,0.60)

  Color separation (palette-based, nearest LAB) **identickou RGB nerozliší** —
  `deduplicate_by_rgb` jednu zahodí (přežijí priority 24/27, masky 40/43 vůbec
  nevzniknou). Disambiguace pak mapovala přeživší priority na jediný symbol s tím
  `inner_color` = `.1` varianta. Detektor tedy lepil `403.1` (vzácná OOM-custom
  "paseka", GT 47) na vše, co bylo většinou `403.0` (rough open land, GT 898).
- **Důkaz, že to není over-detection**: detekováno 1019 bledě žlutých ploch, GT
  (403.0 + 403.1) = 945 → ratio **1.08×**. Celá žlutá: 1482 / 1135 = **1.31×** =
  normální over-segmentace. "21×" vzniklo porovnáním špatně přiřazených kódů.
  Potvrzuje memory `verify-domain-claims-against-source` (nevěř agregátům).
- **`.1` symboly mají v názvu "(upraveno)"** = OOM-custom varianty; `.0` jsou
  standardní ISOM. Uživatel (mapař) potvrdil: rozlišovat `.0` vs `.1` v první
  verzi netřeba, "paseka == louka == open land" je OK výsledek v1.
- **db2omap PoC** (uživatelův dotaz "co už dovedeme"): plný export = Stage 5/7/8,

## Rozhodnutí

- **Brown line codes**: caller (`pic2db.cmd_detect`) parsuje library JEDNOU
  (sdíleno s area v2 disambiguation), resolvuje `(thin_code, thick_code)` přes
  `resolve_brown_line_codes` a předá. Bez `--omap` fallback `"101"`/`"102"`.
- **Area RGB grouping**: `build_priority_to_area_code` seskupuje symboly podle
  **resolved RGB** (ne podle priority indexu) a vybírá základní variantu
  (`_base_variant`, nejnižší ISOM kód). Tím priority 27 → `403.0` místo `403.1`.
- **Ambiguous RGB-skupina** (víc base kódů shodné barvy, např. plná žlutá =
  401/402/412/413/415, color separation NEROZLIŠÍ) → vybrat nejnižší base
  (`401.0` = open land), NE fallback na cizí default. Důvod: nejnižší base =
  nejzákladnější symbol té barvy; fallback na jinou barvu (403.0 pro plnou žlutou)
  by byl horší. Stará ochrana `len(bases)==1 → vynech` odstraněna.
- **Template-aware default**: `resolve_default_area_code` (base z
  `DEFAULT_SYMBOL_PER_CATEGORY` → suffixovaná varianta v library). Slovanka:
  GREEN=406.1, YELLOW=403.0, BLACK=526.0.
- **db2omap PoC = forest sample** (jednodušší georef: `scale="10000"`, local CRS,
  žádný offset; Slovanka má `.pgw` + deklinaci = složitější). Areas i lines jsou
  OMAP `type="1"` (path). Template injekce: regex nahrazení obsahu `<objects>`.
