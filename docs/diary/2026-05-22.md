## Diskuse

- **„3× duplicita" coord-parsingu je ve skutečnosti 2+1** (ověřeno proti zdroji před kódem).
  `omap_mask._parse_coords` (split, drží flag) a `compare_to_omap._object_centroid` (regex
  na jeden `<coords>` text) berou **jeden coords text** → sjednotí se čistě. `georef.
  _compute_coord_bbox` ale skenuje regexem přes **celý `<objects>` blob** (bbox-only, vědomě
  bez full-parse) — jiná operace, ne kopie. TODO popis („3× → 1 parse_coords") byl proto
  optimistický.
- **Po migraci prvních dvou zůstane georef jediným regex skenerem** → duplicita zmizí, aniž
  sáhnu do nejcitlivějšího modulu. Předělávat `_compute_coord_bbox` na ET-parse by přineslo
  marginální DRY za georef-regresní riziko (bbox-fit kalibrace forest Local CRS) → KISS, nechat.
- **`priority == index` je dvojí indexovací schéma vedle sebe.** `get_color(ref)` dělá
  `colors[ref]` (poziční), kde `ref` je **priority** hodnota; `build_category_map` zase
  klíčuje podle `priority`. Mlčky předpokládají, že splývají — když ne, `get_color` tiše vrátí
  špatnou barvu. Invariant ověřen ve 3 reálných (S14), ale kód ho nevaliduje.
- **Realistický scope (poučení ze S14 censure).** `compare_to_omap` split (846 ř., čistě
  strukturní, žádná změna chování) je největší a nejméně urgentní → vlastní sezení, nebundlovat.

## Rozhodnutí

- **`parse_coords(text) -> [(x,y,flag)]` v `omap_parser`** (single source of truth, split-based).
  `omap_mask._parse_coords` smazán → import sdílené; `compare_to_omap._object_centroid`

## Diskuse

- **Hranice řezu „co s čím porovnává", ne „formatter vs ostatní".** Soubor míchá 6 vrstev.
  Klíč: `compare_to_omap` porovnává GT vs **Stage 3 masky** (per category/type — Fáze 0 metrika);
  „review krok 4" porovnává GT vs **DB** (iter_N.json, per ISOM kód). To jsou dva různé účely →
  dva moduly. `format_report` (per-category) zůstává, `format_per_symbol_table` (per-symbol) odchází.
- **Proč i text tabulka, ne jen CSV (širší než doslovné TODO „CSV/review export").** `_gt_by_code`
  a `_symbol_status` jsou sdílené mezi per-symbol tabulkou (vrstva 5) a CSV exportem (vrstva 7).
  Odříznout jen CSV by tyhle privátní helpery rozpůlilo → křížový import `_`-funkcí mezi moduly
  (code smell). Vzít **celý DB↔OMAP per-symbol blok** (5+6+7) = každý modul samostatně koherentní.
- **Cyklus vyřešen lazy importem.** `review_export` top-level `from compare_to_omap import GroundTruth`
  (jen typ). `compare_to_omap.main` importuje review funkce **lazy** uvnitř `--db` bloku → import až
  za běhu (modul už načtený) = jednosměrná závislost, žádný cyklus. Konzistentní se stávajícím stylem
  (`georef`/`DBSnapshot` se taky importují lazy).
- **Žádný modul compare_to_omap neimportuje** (grep) — standalone CLI skript → split nerozbije API.
