# Sezení 2

Pokračování téhož dne. Cíl: Fáze 0 → začít databází symbolů (zadání uživatele).

## Diskuse

- Adresář přejmenován na `Pic2Omap/`, memory dir přesunut — hotovo mimo sezení.
- **Strategie symbolové DB**: namísto manuálního psaní YAML z ISOM PDF (200+ symbolů, riziko překlepů) extrahovat z existujících OMAP souborů. OMAP je XML s plnou definicí symbolů uvnitř — jediný zdroj pravdy, žádná divergence vůči OOM.
- **Architektura ve 2 vrstvách**:
  - Vrstva 1 (`SymbolLibrary`) = rich 1:1 model z OMAP
  - Vrstva 2 (`ColorProfile`, později `SymbolProfile`) = derivovaný model pro detector
- **Paralelní agenti** verifikovali oba OMAP soubory proti IOF PDF spec (ISOM 2017-2 + ISSprOM 2019-2).
- **Klíčové zjištění**: oba OMAP soubory používají **ISOM 2000 / OOM-default schéma**, ne IOF 2017-2 ani ISSprOM 2019-2. Číslování symbolů je posunuté o 1-3 pozice, některé prohozené. Stejný kód = různý symbol napříč spec verzemi (501 = Road v ISOM 2000/2017-2, Paved area v ISSprOM 2019-2).
- **Color separation strategy**: palette-based (fixní paleta z library, nearest in LAB), ne K-means. Deterministické, žádný unsupervised clustering.
- **Anti-aliasing problém**: vrstevnice se rozsekly mezi Brown a OpenOrienteering Orange (LAB distance malá). Řešení: `ColorCategory` vrstva — sloučit blízké barvy do sémantických rodin (BROWN, GREEN, BLACK, YELLOW, BLUE, PURPLE, WHITE, GRAY, RED) podle HSV hue.

## Rozhodnutí

- **DB schéma**: OOM-native, ne IOF přepisování. Pic2Omap bude pracovat s kódy, jak jsou v OMAP (širší kompatibilita s existujícími mapami).
- **Jazyk**: Python (potvrzeno, stdlib + numpy + opencv + skimage).
- **Symbol library**: extrahovat z OMAP parserem, ne ručně z PDF.
- **Color separation**: palette-based, ne K-means.
- **Color category**: HSV hue klasifikace + manuální NAME_OVERRIDES pro hraniční případy (Green 50% Yellow patří k GREEN, ne YELLOW).
- **Spec PDF**: stáhnout pro offline referenci do `docs/`, ale **negitovat** (22 MB, public download). Gitignore.
- **První git commit**: po dokončení dnešního sezení (~1100 řádků kódu + reports).

## Kód

Nové Python moduly (~600 řádků kódu + komentáře):

- `omap_model.py` — dataclassy: `Color`, `SymbolBase` + 5 podtypů (Point/Line/Area/Text/Combined), `SymbolLibrary`, enums (`SymbolType`, `CapStyle`, `JoinStyle`). Jednotky OMAP = mikrometry.
- `omap_parser.py` — XML parser pomocí `xml.etree.ElementTree` (stdlib, žádné lxml). Namespace handling, type dispatch, color reference resolution.
- `color_profile.py` — vrstva 2 nad SymbolLibrary: `ColorProfile` s precomputed LAB hodnotou. `deduplicate_by_rgb()` pro odstranění knockout duplicit.
- `color_category.py` — `ColorCategory` enum + classifier (HSV hue thresholds + manuální overrides). `build_category_map_with_overrides()`.
- `color_separator.py` — Stage 2 pipeline: `separate_colors()` (palette-based, LAB distance), `merge_masks_by_category()`, `make_overview()`, `save_masks()`, `save_category_masks()`.
- `dump_symbols.py` — CLI overview symbolové DB (kolik barev/symbolů per typ, výpis ISOM kódů).
- `separate_demo.py` — CLI color separation: per-priority + per-category masky, overview, report.

Nové dokumenty v `docs/`:

- `dump_forest_sample.txt`, `dump_complete_map.txt` — dump symbolové DB pro oba OMAP soubory.
- `spec_check_ISOM-2017-2.md` (388 řádků) — verifikace forest sample proti IOF ISOM 2017-2 Rev 6 PDF. **Hlavní nález**: OMAP používá ISOM 2000 numbering, ne 2017-2.
- `spec_check_ISSprOM-2019-2.md` (246 řádků) — verifikace complete map proti IOF ISSprOM 2019-2 Rev 6 PDF. **Hlavní nález**: complete map taky používá ISOM 2000 (ne ISSprOM 2019-2), navzdory měřítku 1:4000.
- `ISOM_2017-2_rev6.pdf` (11.7 MB, gitignored), `ISSprOM_2019-2_rev6.pdf` (10 MB, gitignored), `maprunner.pdf` — lokální PDF reference.
- `omap_symbols_list.txt` — kompletní seznam OMAP code+name páry.

Výstupy testu v `output/forest sample/`:

- `priority/` — 16 binárních masek per palette barva.
- `category/` — 8 sloučených masek per ColorCategory.
- `_overview.png` — quantized verze celého obrazu (vizuálně k nerozeznání od originálu).
- `_report.txt` — statistika per barva (% pixelů).

## Příště

1. **Stage 3 — per-category raster ops**:
   - Morfologie: opening/closing pro odstranění šumu (1px artefakty z anti-aliasing)
   - Connected components: rozdělit kategorie na linky / plochy / body
   - Skeletonizace líniových masek → 1px středovky (přípravka pro Stage 5)
2. **Ground truth comparison**: napsat skript, který načte `forest sample.omap` a porovná: kolik vrstevnic je v OMAPu (Brown linie 101/102/103) vs kolik detekuje pipeline z PNG.
3. **Map area extraction**: title "Forest map sample" je nad mapou, zbytečně rozhazuje statistiku. Buď automatický crop (detekce mapového obdélníku), nebo manuální ROI.
4. **ISSprOM template lookup** — pokud sprint je v rozsahu, sehnat oficiální `ISSprOM_2019-2.omap` template z OOM symbol sets distribuce (https://github.com/OpenOrienteering/symbol-sets nebo OOM installdir).
5. **(Bonus) SymbolProfile** — vrstva 2 pro symboly (pro Stage 4). Per-symbol: key features (line width, dashed Y/N, point shape, area pattern).
